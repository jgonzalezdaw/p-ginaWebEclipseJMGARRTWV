Etiquetar

Git tiene la opción de etiquetar (tag) ciertas versiones para encontrarlas más facilmente en el futuro. Lo más común es utilizarlo para etiquetar una cierta versión que ha sido alargada en producción.

Es posible listar los tags disponibles mediante este comando:
git tag

Puedes crear un nuevo tag de la siguiente manera. Con el parámetro -m especificas la descripción del tag.
git tag version1.6 -m 'version 1.6'

Si quieres acceder al código asociado con ese tag:
git checkout tag\_name
