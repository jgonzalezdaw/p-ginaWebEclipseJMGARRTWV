Trabajando con repositorios externos

Creando un repositorio Git externo

Creamos un repositorio remoto en GIT para guardar nuestros archivos en la red y localmente.
Los repositorios que hemos creado no contienen todos los archivos, mientras que los repositorios remotos si, ya que guardan el codigo fuente y el repositorio de GIT, para usar el repositorio que hemos creado añadimos la bandera –bare. -git clone –bare . ../prueba-remota.git

Subir los cambios a otro repositorio - Push

Haremos unos cambios y los subimos al repositorio con los siguientes comandos:
#Modificamos
echo “Primer cambio”
#Hacemos commit
git commit -a -m “Cambios”
#Subimos los cambios
git push ../prueba-remota.git

Add remote

Podemos crear un repositorio con nombre con el comando:
git remote add. Origin, este nombre suele usarse automatica, asi que si clonamos el repositorio, origin indica el repositorio que usamos desde que empezamos el archivo.

#añadimos /prueba-remote.git con origin
git remote add origin ../prueba-repositorio.git
#commit
git commit -a -m “commit repositorio origin”
#subimos origin
git push origin
5.4. Mostrar los repositorios externos existentes
#Ver los repositorios existentes
git remote
5.5. Clonar tu repositorio
Creamos un repositorio nuevo en otro directorio con los comandos siguientes.
#Creamos nuevo directorio
mkdir repositorio nuevo
#Y lo clonamos
git clone ../prueba-remota.git

Trayendo los cambios - Pull

Con pull obtenemos los cambios de otro repositorio, por ejemplo, hacemos un push con un repositorio, y para obtener esos cambios en otro repositorio usamos el pull.
#Primero hacemos unos cambios
echo “cambio Prueba”
#commit
git commit -a -m “CambioPrueba”

#Subimos los cambios

git push origin
#Despues bajamos los cambios en la carpeta que queramos.
Git pull ../prueba-remota-git
#Podemos observar los cambios
less Prueba
